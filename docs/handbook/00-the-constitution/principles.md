# Principles

## 1. The Handbook Leads

Every future product decision starts in the Handbook before it reaches code.

## 2. Product Specs Before Features

New functionality must be described in a Product Spec before implementation.

## 3. Decisions Stay Traceable

Relevant product, design, and engineering decisions must be recorded as ADRs.

## 4. User Clarity Wins

Niva prioritizes comprehension over visual density or cleverness.

## 5. Internal Names Can Be Transitional

The product name is Niva. Internal `Aurora*` component names remain valid until a dedicated technical rename sprint.

## 6. Documentation Does Not Modify Behavior

Documentation sprints do not change screens, logic, integrations, schema, or UI behavior.
