# Analytics

## Overview

Analytics helps the user understand patterns in spending, income, and financial behavior.

## Purpose

Turn transaction history into useful interpretation.

## User Goals

- See where money is going.
- Compare categories.
- Understand trends.
- Identify opportunities to improve.

## Primary Question

What patterns should I notice?

## Information Hierarchy

1. Period and filters.
2. Primary metrics.
3. Charts and category breakdown.
4. Supporting table.
5. Insight summaries.

## Components

- MetricCard.
- Analytics charts.
- Data table.
- Finance filters.
- InsightCard.

## Interactions

- Filter by period, account, category, or type.
- Compare category totals.
- Review insight context.

## States

- Populated analytics.
- Empty data.
- Filtered empty state.
- Loading state.

## Future Ideas

- Financial health score.
- Monthly comparisons.
- Smart anomaly detection.
