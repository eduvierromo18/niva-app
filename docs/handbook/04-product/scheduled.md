# Scheduled

## Overview

Scheduled transactions help the user anticipate upcoming financial events.

## Purpose

Make recurring or future payments visible before they affect the month.

## User Goals

- See upcoming payments.
- Understand due dates.
- Track recurring obligations.
- Avoid missed payments.

## Primary Question

What is coming next?

## Information Hierarchy

1. Upcoming dates.
2. Payment or transaction amount.
3. Account or category context.
4. Status and action.

## Components

- Scheduled transaction dialog.
- Alert card.
- Currency amount.
- Empty state.

## Interactions

- Create scheduled transaction.
- Review upcoming item.
- Update or complete scheduled item.

## States

- Upcoming list.
- No scheduled transactions.
- Due soon.
- Overdue.

## Future Ideas

- Calendar view.
- Monthly cashflow projection.
- Smart reminders.
