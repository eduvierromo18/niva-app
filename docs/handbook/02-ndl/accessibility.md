# Accessibility

## Purpose

Define accessibility expectations for Niva.

## Status

Draft

## Scope

Applies to navigation, forms, charts, tables, modals, feedback, color, keyboard behavior, and text.

## Rules

- Financial information must not rely on color alone.
- Labels and actions must be understandable without context guessing.
- Interactive states must be visible.
- Dense data should remain readable on small screens.
