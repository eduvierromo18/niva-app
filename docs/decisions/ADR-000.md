# ADR-000 Product Operating System

## Decision

Adoptar un Product Operating System dentro del repositorio para documentar vision, milestones, backlog, iteraciones, releases, Product Specs y decisiones.

## Motivo

El proyecto esta evolucionando hacia un producto de software y necesita una metodologia consistente para priorizar, ejecutar y registrar decisiones.

## Consecuencias

- Toda funcionalidad futura debera partir de un Product Spec.
- Las decisiones relevantes deberan documentarse como ADR.
- La documentacion de producto vivira junto al repositorio.
- Esta decision no modifica codigo, pantallas, logica, Supabase ni base de datos.

## Estado

Aceptado
